改 schema 时：
- 迁移即代码：每次变更落一个版本化迁移脚本（沿用项目现有迁移工具/目录），可前滚可回滚，绝不手改生产库。
- 向后兼容分步走：破坏性变更拆成"加新→双写/回填→切读→删旧"多步发布，避免一次性锁表或断服务。
- 加列谨慎：新列优先可空或带默认；给大表加非空列/索引考虑在线 DDL 或分批回填，防长锁。
- 索引配合查询：为高频过滤/排序/连接列建索引；别盲目加，注意写放大与冗余索引。
- 数据迁移幂等：回填/数据修正脚本可重复执行不重复副作用；大批量分批提交。
- 约束与完整性：外键/唯一/非空约束显式声明；删除数据前确认引用关系。
- 迁移与应用同步：新旧代码在迁移窗口内都能工作（扩展-收缩模式），别假设瞬时切换。
